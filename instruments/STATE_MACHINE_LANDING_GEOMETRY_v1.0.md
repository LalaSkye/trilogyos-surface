## States

### 1. SEEKING

- Geometry sensed but not stabilised
- Repeated approach
- High interpretation load
- No reliable footing

### 2. ORBITING

- Recurring contact with the same structure
- Pattern recognised from multiple angles
- Testing, re-entry, comparison
- Truth suspected but not yet trusted

### 3. ALIGNING

- Internal model and external artefacts begin to match
- Less description, more compression
- Fewer contradictory moves
- Routing, language, and build surfaces start converging

### 4. CONTACT

- Geometry no longer theoretical
- Local proofs exist
- Repeated successful touches without collapse
- Can stand briefly, but still checks footing

### 5. LANDED

- Operating from the geometry, not merely toward it
- Architecture supports action
- Fewer re-derivations needed
- State changes begin to inherit structure directly

### 6. BUILD_FROM_GROUND

- Post-landing execution
- Extend, test, instrument, harden
- No need to keep proving the ground every cycle

## Transitions

| From     | To               | Condition                                                        |
|----------|------------------|------------------------------------------------------------------|
| SEEKING  | ORBITING         | When recurrent structure is detected                             |
| ORBITING | ALIGNING         | When separate observations begin converging on the same invariant set |
| ALIGNING | CONTACT          | When artefacts, language, and decisions start matching the geometry |
| CONTACT  | LANDED           | When the geometry becomes load-bearing under real use             |
| LANDED   | BUILD_FROM_GROUND| When effort shifts from locating truth to constructing on top of it |
